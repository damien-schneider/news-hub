<NewsItem
  title="Le code IA explose, les releases ne suivent pas"
  category="ia"
  tags={["étude", "agents", "productivité"]}
  sources={[
    { label: "MIT & Wharton (SSRN)", url: "https://papers.ssrn.com/sol3/papers.cfm?abstract_id=6843118" },
    { label: "via AlphaSignal", url: "https://alphasignal.ai" },
  ]}
>
Sur **100 000 développeurs GitHub** suivis (télémétrie Microsoft, groupe témoin un an plus tôt), les agents async produisent jusqu'à **17×** plus de code que les humains et gonflent les pull requests de **71,8 %** — mais cela ne se traduit que par **+20 %** de releases effectives. Le goulot s'est déplacé de l'écriture vers la revue, les tests et le déploiement, qui restent manuels. Un rapport DORA 2025 le résume : l'IA multiplie les capacités existantes — si la CI et la revue sont déjà lentes, le flux de code généré aggrave l'instabilité.

</NewsItem>

Pour Cutler, sommer les équipes de « montrer plus d'initiative IA » revient à confondre adoption et conformité : on retire l'autonomie, la compétence et l'appartenance — les trois piliers de la motivation selon Deci & Ryan. Sa cible : la course aux métriques de vanité et au narratif « 10×, puis 100× » jamais relié à une hausse de demande réelle. Même l'analyse par fonction des superusers d'[Atlassian](https://www.atlassian.com/blog/ai-at-work/how-to-identify-ai-superusers), plus fine, mesure la capacité créée plutôt que le retour réel.
