<NewsItem
  title="Le bug silencieux du RL multi-tours avec outils"
  category="ia"
  tags={["rl", "tokenization"]}
  sources={[{ label: "Hugging Face", url: "https://huggingface.co/blog/huggingface/tito" }, { label: "via AINews", url: "https://www.latent.space/p/ainews-founders-and-forward-deployed" }]}
>
Décoder la sortie du modèle, parser les appels d'outils, puis re-tokeniser la conversation peut changer les tokens : le gradient s'applique alors à une séquence que le modèle n'a jamais échantillonnée. La règle « Token-In, Token-Out » corrige le tir — ne jamais ré-encoder ce qu'on a décodé, garder un seul buffer de tokens entre les tours.
</NewsItem>
